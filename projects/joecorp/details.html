Step 1: Initial configuration in Sconfig

Upon first boot in Server Core, you're dropped into Sconfig. Change the machine name, ensure activation, check for updates, and reboot. While in Sconfig, make a note of the network adapter index number (option 8) as I will need it for manual IP configuration via PowerShell later.

Step 2: Static IP and DNS in PowerShell

[Add Screenshot Here]

Exit Sconfig (option 15) to go to PowerShell. Use the following commands to set a static IP and DNS server. Since I don't have another DNS server on this network, I'm setting DNS to itself.

New-NetIPAddress -InterfaceIndex [index] -IPAddress [static ip] -PrefixLength 24
Set-DnsClientServerAddress -InterfaceIndex [index] -ServerAddresses [dns server or master dc]
Notes:
  • Most deployments would allow you to do this step within Sconfig. However, it requires the entry of a Gateway's IP address. Since I'm running a private network and don't have a gateway, network configuration needs to be done manually.
  • Once the second dc is up, the primary dns should be the secondary dc's ip with the secondary dns being loopback 127.0.0.1, and vice versa
