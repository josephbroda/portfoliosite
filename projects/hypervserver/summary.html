Remote Proxmox Server

A low cost virtualization platform built on reliability, strategic resource optimization, and out of band management.

Overview

The Problem

Running Hyper-V on my daily driver PC created significant friction:

  • Resource Contention: Spinning up the lab required shutting down other applications.
  • Heat & Noise: Leaving a high-performance desktop on 24/7 in a dorm room was impractical. (heat and noise)
  • Disrupted Workflow: Restarting VMs for every session discouraged long-term projects.

The Objective

I needed a dedicated, off-site server to host my VMs 24/7. The key requirements were:

  • Reliability: Stable hardware and data protection.
  • Efficiency: Low power consumption and heat output.
  • Cost-Effectiveness: Utilizing existing components where possible.
  • Remote Management: Full control without physical access.

The Solution

I built a custom server using a mix of repurposed hardware and strategic new and used purchases, focusing on reliability and remote manageability while also keeping costs down.


Hardware & Storage

  • Compute: Ryzen 9 3950x (16-core) on an ASRock B550m Pro 4 motherboard.
  • Memory: 64GB (4x16GB) ECC Unbuffered DDR4.
  • Storage: NVMe SSD for OS, 3x 1TB NAS SATA SSDs for VMs in RAIDZ1 for redundancy and space efficiency.

Backup & Reliability

  • Proxmox Backup Server: Virtualized on the main host with 2x 4TB HDDs passed through directly for incremental backups, facilitating easy migration to bare metal in the future.
  • Power Protection: UPS with NUT server/client configuration for synchronized graceful shutdowns.
  • Monitoring: Homebridge with APC UPS plugin for uptime and power status in Apple Home.

Network & Management

  • Remote Access: JetKVM via Tailscale for secure Out-of-Band (OOB) management of UEFI and power.
  • Security: Isolated LAN for all lab traffic to separate it from the home network.

Total Cost: Under $1000 going off of market prices as of November 2025, delivering enterprise-grade features at a fraction of the price of used servers.

Core Accomplishments

  • Resilient Backup Strategy: Virtualizing Proxmox Backup Server with direct disk passthrough to combine the flexibility of virtualization with the data portability of physical hardware.
  • Out-of-Band Management (OOB): Deploying and configuring a secure, hardware-level KVM over IP solution via JetKVM and Tailscale for remote access and troubleshooting, ensuring high management uptime.
  • Strategic Resource Optimization: Utilizing Cost-Benefit Analysis to prioritize budget towards high-impact components, reusing existing capable hardware when available, and taking advantage of the used market when appropriate to maximize performance per dollar.
  • Virtualization Platform Management: Deployment and configuration of Proxmox VE for a simplified, non-domain management experience that is accessible from any device.
  • Automated Power Management: Configured Network UPS Tools (NUT) in a server/client architecture to orchestrate graceful shutdowns across the host and backup server during power outages, with real-time status monitoring integrated into Apple Home via Homebridge.
  • Network Segmentation: Implemented a dedicated LAN for lab traffic to isolate it from the main home network, enhancing security and reliability.

Lessons Learned: Hyper-V Server & Workgroups

I originally provisioned this project with Hyper-V Server 2019, hoping Windows Admin Center would provide sufficient web-based management, as well as giving me more hands-on experience with Windows administration. However, running in a workgroup environment led to persistent WinRM and CredSSP authentication issues. Even after TrustedHost and Group Policy workarounds, the VMConnect console on Windows Admin Center remained non-functional, making remote management from my prefered platform, macOS, impractical. This experience drove the switch to Proxmox VE for its robust, platform-agnostic web interface.
